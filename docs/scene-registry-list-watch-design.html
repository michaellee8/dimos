Scene Registry

A Kubernetes-style List/Watch model for scene state: sources assert spatial entities, consumers take a versioned snapshot, then follow ordered deltas. Planning correctness comes from an explicit WorldMonitor sync barrier before motion planning.

Primary analogy: Kubernetes List/Watch Robotics analogy: MoveIt PlanningScene + Tesseract Environment v1: inline payloads No memory2 dependency

3. Proposed data model

A SceneEntity is a broad spatial item, not necessarily a semantic object. Payload type tells consumers how to interpret it.

SceneEntity { id: string // source-scoped stable id source_id: string frame_id: string // e.g. "world" pose: Pose // identity for world-frame maps payloads: ScenePayload[] // inline in v1 updated_at: float } ScenePayload { type: string // PointCloud2, Box3D, Mesh, TSDF, ... data: typed inline payload } SceneSnapshot { version: int entities: SceneEntity[] } SceneEntityDelta { version: int operation: add | update | remove entity_id: string entity?: SceneEntity }

4. v1 simplifications

Inline payloads only

No memory2 refs, stream refs, or external payload handles in v1.

One aggregate voxel-map entity

Do not explode voxels into per-cell SceneEntities.

Voxel resolution via config

For now, follow existing module practice: voxel size is configured on producing/consuming modules.

Future improvement

Add a typed voxel-map payload carrying resolution and occupancy metadata with the payload.

7. Why this avoids the old sync bug

The planner no longer relies on incidental perception-side caches. Before every relevant plan, WorldMonitor takes a coherent registry snapshot and applies planner-specific projections.

Old failure shape: grasp generator sees pointcloud context planner world misses obstacles trajectory collides New shape: SceneRegistry snapshot at version N WorldMonitor syncs version N into RoboPlanWorld planner checks trajectory against projected obstacles

8. Versioning model

What the version means

registry_version is a global monotonic cursor for registry mutations. Every accepted source snapshot or delta that changes current state advances the version. A snapshot is a coherent view of all entities at one version.

v41: simulation/mujoco/table ADDED v42: mapping/manipulation_voxel_map UPDATED v43: perception/object_7 ADDED get_snapshot() -> SceneSnapshot(version=43, entities=[...]) watch(from_version=43) -> events strictly after v43

Why consumers care

Versions let consumers say exactly which scene they rendered, cached, or planned against. They are not wall-clock freshness guarantees; they are consistency cursors.

Planner reports planned_against_version Visualizer knows whether it is caught up Watch gaps recover by relisting Debug logs can reproduce scene state
List/Watch contract: 1. Consumer calls get_snapshot() 2. Consumer records snapshot.version 3. Consumer watches from that version 4. If watch is interrupted or too old, consumer relists Planning contract: 1. WorldMonitor calls get_snapshot() 2. WorldMonitor projects that exact version into WorldSpec 3. Planner produces a plan tagged with that version 4. Executor may reject/replan if registry changed past policy threshold

9. Consumer workflows

Viser startup

Initial scene render

  • Call get_snapshot()
  • Render every supported payload
  • Start watch(snapshot.version)
Viser live

Prompt visual updates

  • add: add layer/object
  • update: replace payload render
  • remove: delete render node
Plan request

Sync before planning

  • WorldMonitor snapshots registry
  • Projects supported payloads
  • Applies exclusion policy
Planning

Tagged result

  • Plan against synced WorldSpec
  • Return path + scene version
  • Expose projected obstacle summary
Source update

Source snapshot apply

  • Source asserts current entities
  • Registry diffs per source
  • Missing entities become removals
Example consumer behavior: Visualizer: snapshot = registry.get_snapshot() render(snapshot.entities) for delta in registry.watch(snapshot.version): apply_visual_delta(delta) WorldMonitor: snapshot = registry.get_snapshot() result = sync_planning_scene(snapshot, policy={exclude: current_target}) plan = manipulation_planner.plan(world_version=result.registry_version) Source adapter: entities = build_entities_from_current_source_state() registry.apply_source_snapshot(source_id, entities)
