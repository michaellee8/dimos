---
title: "Adding a New Arm"
---

Complete build procedure for integrating a manipulator arm into DimOS. This is the reference version, written for an agent. Humans usually want [Adding a Custom Arm](/docs/capabilities/manipulation/adding_a_custom_arm.md) instead.

You write four files: two describing the hardware, two describing the robot. Nothing else in DimOS changes.

## Read these first

Do not transcribe from this page what you can read from the code.

| File | Why |
|------|-----|
| `dimos/hardware/manipulators/spec.py` | The `ManipulatorAdapter` Protocol. Authoritative signatures. 26 methods. |
| `dimos/hardware/manipulators/mock/adapter.py` | Complete reference implementation of all 26. Copy its structure. |
| `dimos/hardware/manipulators/a750/adapter.py` | Smallest real adapter. Vendor SDK, gripper, serial. |
| `dimos/robot/manipulators/a750/config.py` | Smallest real config. |
| `dimos/robot/manipulators/a750/blueprints/teleop.py` | Smallest real blueprint. |
| `dimos/robot/manipulators/common/blueprints.py` | The helpers you must use. |
| `dimos/control/components.py` | `HardwareComponent`, `make_joints`, `split_joint_name`. |
| `dimos/manipulation/planning/spec/config.py` | `RobotModelConfig`. Only if the arm needs motion planning. |

If the arm has no vendor Python SDK and speaks a raw bus, read `dimos/hardware/manipulators/openarm/`, which drives SocketCAN directly.

## Invariants

These are not derivable from the code. Each one, violated, produces either a hard crash or a silent no-op. They are the reason this page exists.

**1. Joint names are slash-separated.** The coordinator recovers the owning `hardware_id` from the prefix.

```python session=arm
from dimos.control.components import make_joints, split_joint_name

print(make_joints("arm", 6))
print(split_joint_name("arm/joint1"))
```

Hand-writing `arm_joint1` instead raises:

```python session=arm expected-error
split_joint_name("arm_joint1")
```

Always call `make_joints`. Never build the list by hand.

**2. No `__init__.py`, anywhere.** `dimos/hardware/manipulators/` and `dimos/robot/manipulators/` are PEP 420 namespace packages and contain none. Import submodules directly, never from the package.

**3. Blueprints must be assigned from `autoconnect(...)` or `.blueprint(...)`.** `dimos/robot/all_blueprints.py` is generated by an AST scanner that recognises only those two. A bare `x = coordinator(...)` builds a valid blueprint that is never registered and gets no CLI name. Wrap single-module blueprints in `autoconnect(...)` even when it looks redundant.

**4. `coordinator_task_name` must equal a `TaskConfig.name`.** `ManipulationModule` executes trajectories via `task_invoke(coordinator_task_name, "execute", ...)`, and `trajectory_task(hw)` names its task `traj_{hardware_id}`. If they disagree, planning succeeds, execution returns `None`, and the arm does not move. No exception, no error state.

**5. The adapter registry has a golden set.** `EXPECTED_NAMES["manipulators"]` in `dimos/hardware/test_adapter_registries.py` pins the exact set of registered names, and CI fails until you add yours. If the vendor SDK is an optional dependency, also add its top-level module name to `OPTIONAL_VENDOR_MODULES` in the same file.

**6. Adapter constructors must accept `**_: object`.** The coordinator passes `dof=`, `address=`, `hardware_id=`, plus every key in `adapter_kwargs`.

**7. Everything crossing the adapter boundary is SI.** Radians, meters, rad/s, Nm, Newtons.

**8. Unsupported features return, they never raise.** `None` from a read, `False` from a write. Absent velocity or effort feedback returns zeros.

**9. `_registry.py` is stdlib-only.** It is imported on machines without the vendor SDK, so the name still appears in `available()` and a missing SDK fails loudly at `create()`.

**10. Two unrelated `_registry.py` families exist.** `dimos/hardware/**/_registry.py` declares `ADAPTER_FACTORIES` and is the one you write. `dimos/control/tasks/**/_registry.py` declares task factories and routing cards, and is only for adding a new control *task type*. Adding an arm never touches it.

## File 1 — the adapter

`dimos/hardware/manipulators/<arm>/adapter.py`. Duck-typed against the Protocol. No inheritance, no base class.

```python skip
from __future__ import annotations

from dimos.hardware.manipulators.spec import ControlMode, JointLimits, ManipulatorInfo
from dimos.utils.logging_config import setup_logger

logger = setup_logger()


class YourArmAdapter:
    def __init__(
        self,
        address: str = "192.168.1.100",
        dof: int = 6,
        initial_positions: list[float] | None = None,
        **_: object,          # required — see invariant 6
    ) -> None:
        self._address = address
        self._dof = dof
        self._positions = list(initial_positions) if initial_positions else [0.0] * dof
        self._sdk = None
        self._control_mode = ControlMode.POSITION

    def connect(self) -> bool:
        try:
            from yourarm_sdk import YourArmSDK      # lazy — SDK is an optional dep
            self._sdk = YourArmSDK(self._address)
            self._sdk.connect()
            return bool(self._sdk.is_alive())
        except ImportError:
            logger.error("yourarm-sdk not installed")
            return False
        except Exception as e:
            logger.error(f"connect failed at {self._address}: {e}")
            return False

    # ... the remaining 25 methods. Copy the structure from mock/adapter.py.
```

Import the vendor SDK inside `connect()`, not at module scope, whenever it is an optional dependency. A module-scope import breaks `import dimos` on machines without the arm.

## File 2 — the adapter manifest

`dimos/hardware/manipulators/<arm>/_registry.py`. Stdlib imports only. Nothing else in the file.

```python skip
ADAPTER_FACTORIES = {
    "yourarm": "dimos.hardware.manipulators.yourarm.adapter:YourArmAdapter",
}
```

## File 3 — the robot config

`dimos/robot/manipulators/<arm>/config.py`. Two factories: one describes the arm to the coordinator, one to the planner.

```python skip
from __future__ import annotations

import math

from dimos.control.components import HardwareComponent, HardwareType, make_joints
from dimos.core.global_config import global_config
from dimos.manipulation.planning.spec.config import RobotModelConfig
from dimos.robot.manipulators._modeling import base_pose, coordinator_joint_mapping, joint_names
from dimos.utils.data import LfsPath

YOURARM_HOME_JOINTS = [0.0, 0.0, -math.radians(90), 0.0, 0.0, 0.0]
YOURARM_MODEL_PATH = LfsPath("yourarm_description/urdf/yourarm.urdf")
YOURARM_PACKAGE_PATHS = {"yourarm_description": LfsPath("yourarm_description")}


def make_yourarm_hardware(
    hw_id: str = "arm",
    *,
    adapter_type: str = "yourarm",
    address: str | None = None,
    gripper: bool = True,
    auto_enable: bool = True,
    home_joints: list[float] | None = None,
) -> HardwareComponent:
    return HardwareComponent(
        hardware_id=hw_id,
        hardware_type=HardwareType.MANIPULATOR,
        joints=make_joints(hw_id, 6),                     # invariant 1
        adapter_type=adapter_type,                         # must match the ADAPTER_FACTORIES key
        address=address,
        auto_enable=auto_enable,
        gripper_joints=[f"{hw_id}/gripper"] if gripper else [],
        adapter_kwargs={"initial_positions": home_joints or YOURARM_HOME_JOINTS},
    )


def yourarm_hardware(hw_id: str = "arm", *, mock_without_address: bool = False) -> HardwareComponent:
    """Real hardware when an address is configured, mock otherwise."""
    address = global_config.device_path
    if mock_without_address and not address:
        return make_yourarm_hardware(hw_id, adapter_type="mock", address=None)
    return make_yourarm_hardware(hw_id, address=address or "/dev/ttyACM0")


def make_yourarm_model_config(
    name: str = "arm",
    *,
    coordinator_task_name: str | None = None,
) -> RobotModelConfig:
    dof = 6
    return RobotModelConfig(
        name=name,
        model_path=YOURARM_MODEL_PATH,
        base_pose=base_pose(),                             # base_pose(y=0.5) offsets a second arm
        joint_names=joint_names(dof),                      # URDF names: joint1..joint6
        end_effector_link="gripper_base",
        base_link="base_link",
        package_paths=YOURARM_PACKAGE_PATHS,
        auto_convert_meshes=True,
        collision_exclusion_pairs=[],                      # start empty; add what the planner reports
        joint_name_mapping=coordinator_joint_mapping(name, dof),        # "arm/joint1" -> "joint1"
        coordinator_task_name=coordinator_task_name or f"traj_{name}",  # invariant 4
        gripper_hardware_id=name,
        home_joints=YOURARM_HOME_JOINTS,
    )
```

`gripper_joints` routes those joints through the adapter's `read/write_gripper_position` rather than the normal joint path. `adapter_kwargs` is an untyped passthrough to your constructor. `joint_name_mapping` bridges the namespaces: the coordinator knows `arm/joint1`, the URDF knows `joint1`.

Skip `make_yourarm_model_config` entirely if the arm needs no motion planning. Control and teleop work without a URDF.

URDFs ship through Git LFS. `LfsPath` is a `Path` subclass that defers the download until the path is read, so importing a blueprint does not fetch data. Do not stringify an `LfsPath` at import time; that triggers the download.

## File 4 — the blueprints

`dimos/robot/manipulators/<arm>/blueprints/basic.py`. Note `blueprints` is a package directory, not a single module. Real arms split it into `basic.py`, `teleop.py`, `planner.py`, `perception.py`.

```python skip
from __future__ import annotations

from dimos.core.coordination.blueprints import autoconnect
from dimos.robot.manipulators.common.blueprints import coordinator, planner, trajectory_task
from dimos.robot.manipulators.yourarm.config import make_yourarm_model_config, yourarm_hardware

_hw = yourarm_hardware("arm", mock_without_address=True)

coordinator_yourarm = autoconnect(                          # invariant 3 — do not unwrap
    coordinator(hardware=[_hw], tasks=[trajectory_task(_hw)]),
)

yourarm_planner_coordinator = autoconnect(
    planner(robots=[make_yourarm_model_config(name="arm")]),
    coordinator(hardware=[_hw], tasks=[trajectory_task(_hw)]),
)
```

`autoconnect` matches output ports to input ports by type and name, so the planner's `coordinator_joint_state` input finds the coordinator's output with no explicit topic wiring.

Task helpers, by control mode:

| Helper | `type` | For |
|--------|--------|-----|
| `trajectory_task` | `trajectory` | Executing planned joint trajectories |
| `cartesian_ik_task` | `cartesian_ik` | Cartesian pose commands |
| `eef_twist_task` | `eef_twist` | End-effector twist, e.g. keyboard teleop |
| `teleop_ik_task` | `teleop_ik` | VR / Quest teleop |

`cartesian_ik_task` and `eef_twist_task` run IK inside the coordinator and additionally require `model_path=` and `ee_joint_id=`. See `dimos/robot/manipulators/a750/blueprints/teleop.py`.

## Verify

Run in order. Each must pass before the next. The snippets below use the shipped `mock` adapter so they execute as written; substitute your own name when checking your arm.

**1. The adapter is discovered and satisfies the Protocol.**

```python session=arm
from dimos.hardware.manipulators.registry import adapter_registry
from dimos.hardware.manipulators.spec import ManipulatorAdapter

print(adapter_registry.available())

adapter = adapter_registry.create("mock", dof=6)
print("satisfies Protocol:", isinstance(adapter, ManipulatorAdapter))
```

`isinstance` against the Protocol is the cheapest way to catch a misspelled or missing method, because the Protocol is `runtime_checkable`. Use it.

**2. Joint names and task names line up.**

```python session=arm
from dimos.robot.manipulators.a750.config import make_a750_model_config
from dimos.robot.manipulators.common.blueprints import trajectory_task
from dimos.robot.manipulators.a750.config import a750_hardware

hw = a750_hardware("arm", mock_without_address=True)
print("joints:", hw.joints)
print("gripper:", hw.gripper_joints)
print("task name matches coordinator_task_name:",
      trajectory_task(hw).name == make_a750_model_config().coordinator_task_name)
```

**3. CI gates.**

```sh skip
pytest dimos/hardware/test_adapter_registries.py        # fails until the golden set is updated
pytest dimos/robot/test_all_blueprints_generation.py    # regenerates dimos/robot/all_blueprints.py
```

The generation test fails the first time by design. It rewrites `all_blueprints.py`, then fails because the file now has uncommitted changes. Commit the regenerated file and re-run.

**4. The blueprint actually registered.**

```sh skip
grep yourarm dimos/robot/all_blueprints.py
```

No match means you violated invariant 3.

**5. It runs.**

```sh skip
dimos run coordinator-yourarm      # mock adapter first, then real hardware
```

## Failure modes

| Symptom | Cause |
|---------|-------|
| `ValueError: Joint name 'arm_joint1' missing separator '/'` | Invariant 1. Hand-written joint names. |
| `dimos run` cannot find the blueprint | Invariant 3. Assigned from a bare helper call. |
| Blueprint missing from `all_blueprints.py` | Invariant 3. |
| Planning succeeds, arm does not move, no error | Invariant 4. `coordinator_task_name` matches no task. |
| `test_declared_names_match_golden_set` fails | Invariant 5. |
| `TypeError: __init__() got an unexpected keyword argument 'hardware_id'` | Invariant 6. Add `**_: object`. |
| `isinstance(adapter, ManipulatorAdapter)` is `False` | A Protocol method is missing or misspelled. Diff against `spec.py`. |
| `FileNotFoundError: ... data/.lfs/...` | An `LfsPath` was stringified at import, or the archive name is wrong. |
| `import dimos` breaks without the arm attached | Vendor SDK imported at module scope. Move it into `connect()`. |

## Do not

- Create `__init__.py`.
- Hand-edit `dimos/robot/all_blueprints.py`. It is generated.
- Modify `dimos/hardware/manipulators/spec.py` or `registry.py`.
- Add a card to `dimos/control/tasks/**/_registry.py`. That is for new task types, not new arms.
- Raise from an adapter method for an unsupported feature. Return `None` or `False`.
